Welcome

This project is to help demonstrate your ability to implement a small feature with with little direction in how you're going to accomplish it. You will flex your layout, CSS/SCSS, JavaScript, and data handling muscles and show your creativity both in interface design and programming.

The Scenario

Our members are learning the world of ultrasonography. Each of our members receive a probe that gives them access to our software and various modules to practice scanning.

You will create a Probe Status page that our Operations team can use to keep an eye on probes and their expirations.

The Task

We've already created a route and component for you to start building out your feature: Probes

It is up to you to display all of the data from the provided JSON files. Here is a list of target objectives:

  • Lay out the data so it is readable and easy on the eyes. We want our ops team to be able to sift throught this data and look for any probes that might need attention.
  • Organize your code, write your own CSS/SCSS, create new components/files, whatever you need to do to make the code easy to read and ultimately maintain.
  • Use all of the data in the available JSON files. We want to know the unique probe ID, the status, who it belongs to, and what modules it has licensed.

Extra Credit

Add in an overview based on the data within the JSON files.

  • Total number of probes
  • Number of active vs expired probes
  • A tally of how many probes each module has access to
  • etc.

The Data

You can make HTTP requests to the following endpoints to get JSON formatted data to use in your project:

  • https://sonosim.com/candidates/probes.json
  • https://sonosim.com/candidates/modules.json

If for whatever reason you have trouble making an HTTP request to the available endpoints, you will find these same JSON files in this project's assets directory
